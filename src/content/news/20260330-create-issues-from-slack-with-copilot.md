## Overview

GitHub added a small but very real workflow feature on March 30: teams can now create GitHub Issues directly from Slack by mentioning `@GitHub` and describing the work in natural language. The integration can generate structured issues, create sub-issues, and let users refine the result inside the thread before creation.

It is not a giant platform launch, but it is a concrete example of AI moving closer to the messy handoff points where engineering work actually starts.

## Why it matters

A lot of software work begins in chat, not in a tracker. Requirements, bugs, and follow-ups often get spotted in Slack and then die there because converting a conversation into usable work items is annoying. This feature matters because it reduces that friction at the exact place where context is freshest.

It also fits the broader trend toward agentic software tooling that sits inside existing collaboration surfaces instead of forcing people into a separate AI-first app.

## Key details

- The GitHub app for Slack can create structured issues from natural-language prompts in a channel.
- It can also generate sub-issues so teams can break larger requests into a parent-child hierarchy in one pass.
- Users can iterate in conversation mode within a thread before committing the final issue.
- Channel-level settings allow default repositories for issue and pull-request creation.
- GitHub says the feature is available with all Copilot plans through the GitHub app in Slack.

## Industry signal

The interesting part is not issue creation by itself. It is the workflow placement. AI developer tooling keeps moving upstream from code generation into work intake, planning, and coordination. That makes this a better pick than another routine IDE release note because it speaks to how engineering operating systems are changing, not just editor features.
