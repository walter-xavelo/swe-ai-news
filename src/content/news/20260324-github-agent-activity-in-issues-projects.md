## Overview

GitHub has added agent-session visibility directly to Issues and Projects. When a coding agent such as Copilot, Claude, or Codex is assigned to an issue, the session now appears under the assignee with a live status like `queued`, `working`, `waiting for review`, or `completed`.

## Why It Matters For SWE

This is a meaningful workflow shift. AI agents are no longer treated as sidecar tools living only in terminals or chats; they are becoming first-class participants in engineering planning systems. That makes it easier for teams to coordinate autonomous work without losing visibility.

## Key Signals

- Agent sessions now appear natively in GitHub Issues sidebars.
- Project table and board views can display attached agent sessions and status.
- Teams can monitor progress across many agent-driven tasks without leaving planning views.
- Developer platforms are starting to treat AI agents as trackable workers, not just assistants.

## SWE Industry Takeaway

The more interesting trend here is operational, not flashy. Once agent execution is visible inside project management surfaces, teams can start managing AI work the same way they manage human work: assign it, monitor it, review it, and move it through delivery flows.
