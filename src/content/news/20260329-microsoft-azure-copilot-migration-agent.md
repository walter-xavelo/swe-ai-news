## Overview

Microsoft has introduced Azure Copilot Migration Agent as an AI-powered assistant embedded in the Azure portal. Rather than executing migrations directly, the tool focuses on the planning phase: estate discovery, dependency analysis, landing-zone preparation, and migration sequencing.

## Why it matters

A lot of enterprise migration pain sits before the first workload actually moves. Planning, dependency mapping, and modernization handoffs are slow, fragmented, and expensive. Microsoft is betting that AI can compress that front-end effort and make Azure Migrate feel more like an intelligent operator than a reporting dashboard.

## Key details

- The agent works on top of Azure Migrate data.
- It can support agentless discovery of VMware estates and produce inventory, dependency maps, and 6R recommendations.
- It can help generate landing-zone artifacts aligned with Microsoft’s Cloud Adoption Framework.
- It can hand modernization work toward development workflows, including GitHub Copilot-assisted paths for application upgrades.

## The catch

The tool is best understood as a planning copilot, not an end-to-end migration engine. Replication, test migration, and cutover still happen through existing Azure Migrate workflows. Even so, this is a meaningful step in turning cloud migration into an agent-assisted engineering process rather than a pile of manual assessment documents.
